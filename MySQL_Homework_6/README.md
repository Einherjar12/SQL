Тема: Использование баз данных. Часть 1.

Задание 1.
Создайте трёхтабличную базу данных Sales(продажи). В этой базе данных должны быть следующие
таблицы Sales (информация о конкретных продажах), Salesmen (информация о продавцах),
Customers (информация о покупателях). Создайте приложение для отображения данных из таблиц.
Меню приложения должно содержать такой минимальный набор отчётов:
- Отображение всех сделок;
- Отображение сделок конкретного продавца;
- Отображение максимальной по сумме сделки;
- Отображение минимальной по сумме сделки;
- Отображение максимальной по сумме сделки для конкретного продавца;
- Отображение минимальной по сумме сделки для конкретного продавца;
- Отображение максимальной по сумме сделки для конкретного покупателя;
- Отображение минимальной по сумме сделки для конкретного покупателя;
- Отображение продавца, у которого максимальная сумма продаж по всем сделкам;
- Отображение продавца, у которого минимальная сумма продаж по всем сделкам;
- Отображение покупателя, у которого максимальная сумма покупок по всем сделкам;
- Отображение средней суммы покупки для конкретного покупателя;
- Отображение средней суммы покупки для конкретного продавца.

Тема: Использование баз данных. Часть 2.

Задание 1.
Для базы данных Больница, которую вы разрабатывали в рамках курса «Теория Баз Данных» создайте
приложение для взаимодействия с базой данных. Этот интерфейс должен позволять:
- Вставку строк в таблицы базы данных;
- Обновление строк в таблицах базы данных. При попытке обновления всех строк в одной таблице нужно выдавать запрос на
подтверждение пользователю. Обновлять все строки, можно только после положительного подтверждения пользователем;
- Удаление строк из таблиц баз данных. При попытке удаления всех строк в одной таблице нужно выдавать запрос на
подтверждение пользователю. Удалять все строки, можно только после положительного подтверждения пользователем;
- Создавать отчеты:
  • Вывести полные имена врачей и их специализации,
  • Вывести фамилии и зарплаты (сумма ставки и надбавки) врачей, которые не находятся в отпуске,
  • Вывести названия палат, которые находятся в конкретном отделении,
  • Вывести названия отделений без повторений, которые спонсируются конкретной компанией,
  • Вывести все пожертвования за указанный месяц в виде: отделение, спонсор, сумма пожертвования, дата пожертвования,
  • Вывести фамилии врачей с указанием отделений, в которых они проводят обследования.

Тема: Использование баз данных. Часть 3.

Задание 1.
Создайте приложение «Социальная сеть». Приложение должно хранить информацию о пользователе, его друзьях,
постах пользователя. Возможности приложения:
- Вход по логину и паролю;
- Добавление пользователя;
- Удаление пользователя;
- Редактирование информации о пользователе;
- Поиск пользователя по ФИО;
- Просмотр информации о пользователе;
- Просмотр всех друзей пользователя;
- Просмотр всех постов пользователя.
Данные необходимо хранить в NoSQL базе данных. В качестве платформы можно использовать MongoDB.

--------------------------------------------------

Задание 2.
Создайте приложение «Литературный музей». Приложение должно хранить информацию об экспонатах,
людях, имеющих отношение к экспонатам. Возможности приложения:
- Вход по логину и паролю;
- Добавление экспоната;
- Удаление экспоната;
- Редактирование информации об экспонате;
- Просмотр полной информации об экспонате;
- Вывод информации о всех экспонатах;
- Просмотр информации о людях, имеющих отношение к конкретному экспонату;
- Просмотр информации об экспонатах, имеющих отношение к конкретному человеку;
- Просмотр набора экспонатов на основе некоторого критерия. Например, показать все книжные экспонаты.
Данные необходимо хранить в NoSQL базе данных. В качестве платформы можно использовать MongoDB.

--------------------------------------------------

Задание 3.
Создайте приложение «Записная книжка». Приложение должно хранить информацию о заметках пользователя.
Возможности приложения:
- Вход по логину и паролю;
- Добавление заметки;
- Удаление заметки;
- Редактирование заметки;
- Просмотр заметки. Если заметка состоит из нескольких частей, нужно отобразить все части;
- Показ всех заметок;
- Просмотр заметок за определенный период времени;
- Отображение заметок, содержащих набор заданных слов.
Данные необходимо хранить в NoSQL базе данных. В качестве платформы можно использовать MongoDB.
